# Оркестрація Навичок (Skills Orchestration)

## Правило Сесій (Memory Manager)

Навичка `memory-manager` — це наскрізний процес, який супроводжує ВСЮ змістовну роботу:
- **Початок роботи:** Якщо користувач не ініціював `/db` сам — нагадай йому: "Хочете розпочати сесію (`/db`), щоб зберегти контекст цієї роботи?"
- **Під час роботи:** Після кожного значущого етапу (завершення інтерв'ю, створення плану, написання коду) — запропонуй зберегти проміжний результат через `/sa`.
- **Завершення:** Перед закінченням діалогу — запропонуй фіналізувати сесію через `/ss`.

> **КРИТИЧНО:** Не забувай про memory-manager. Це найчастіша помилка.

## Ланцюжки Навичок (Skill Chains)

### Новий проект / продукт / фіча

```
discovery-interview → project-planner → Розробка (кодинг)
```

1. **`discovery-interview`** — Зібрати вимоги, з'ясувати контекст, створити специфікацію.
2. **`project-planner`** — На основі специфікації: декомпозиція на задачі, фази, таймлайни.
3. **Розробка** — Писати код відповідно до плану.

### Створення нової навички (Skill)

```
discovery-interview → project-planner → skill-creator
```

1. **`discovery-interview`** — Зрозуміти, що саме навичка повинна робити, для кого, які ресурси потрібні.
2. **`project-planner`** — Спланувати структуру навички, визначити файли та залежності.
3. **`skill-creator`** — Створити навичку за планом.

### Покращення існуючого (рефакторинг, баг-фікс, невелика фіча)

Якщо задача чітка і зрозуміла — НЕ потрібен повний ланцюжок:
- Якщо є сумніви щодо обсягу → `discovery-interview` (скорочений)
- Якщо обсяг зрозумілий, але задача велика → `project-planner`
- Якщо задача проста → одразу до роботи

### Простий запит / питання

Не потребує ланцюжка — відповідай напряму.

## Правила Маршрутизації

1. **НІКОЛИ** не переходь до `project-planner` без специфікації (або від `discovery-interview`, або наданої користувачем).
2. **НІКОЛИ** не починай створювати навичку через `skill-creator` без розуміння, що вона має робити.
3. Якщо користувач каже "зроби проект" / "нова ідея" / "хочу побудувати" — починай з `discovery-interview`.
4. Якщо користувач каже "сплануй" / "розбий на задачі" і вже є специфікація — починай з `project-planner`.
5. Якщо специфікації немає — спершу запитай або запусти `discovery-interview`.
